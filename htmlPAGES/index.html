ICS Data Visualization Project 3: We are family

About:

The page features two display windows, the upper one is set by default to a pie chart, the lower to a filtered table. The display windows are changed via the input controls on the left of the image displayed below.

The controls work in a dynamic and dependant fashion. There are two lists populated with options, the upper list called "Scope" defines what is what the fields sorted from the second list "filter by" will be grouped by.
Example 1: "Scope: Seasons, Filter by: Bart Simpson" we shall tabulate the amount of appearances Bart Simpson has in each season.
Example 2: "Scope: Season 2, Filter by: Homer Simpson" will display the count of appearances of Homer by episodes within that season.
Example 3: "Scope: Homer Simpson, Filter by: Seasons" will display in how many seasons Homer was seen, so if there are 20 seasons, the result would be 20. Which is a bit pointless in itself, but using multiple selections you will e.g. "Scope: Homer, Lisa, etc, Filter by: Seasons" will result in the characters that are recurring in each season. THe most practical approach to this matter would be: "Scope: Homer, Lisa, etc, Filter by: Season 3" which would tell us how many episodes featured each of the characters within the season.
Each list is by default set to display the characters in order by population but as shown in the image above there is a flag which switches between alphabetical and appearance based ordering. The only other "flag" on the search controls is at the top which switches terms between image and textual. Search bar functionality will be implemented as well unless space constraints dictate otherwise. Another option barring space constrictions will be a "display" window where things you select will have their portrait populated to visually represent your filter options, it is the window towards the top with the words "Name at top" inside of its borders.
Moving down once more to the "Filter Threshold" we have another filter option for the user to use. As per the requirement on the project 3 page, you have the ability for quantities exceeding a certain % or Quantity to not appear on your generated charts. Meaning if you want to do: "Scope: Characters (all), Filter by: Season 4" and set the threshold to min= 2 and max=10 major characters would be excluded from the chart but so would characters who only appear once in season 4.
The window at the bottom left is really a list which is populated by past search queries, if you wish to iterate back and forth between a couple queries and compare the data generated. On the left side of the default table and chart, we see a sub window with labels: "C1, C2, C3, C..., Cn, T" allows that window to display the filtered data via a certain chart or table. Last but not least, the Exclusive or indicates only one or the other in the Scope may be selected. Based upon the selection the filter by either gets (based on Characters) || (based on seasons) thus the "Seasons | Characters" and the mirrored options "Voice actor | Voice actor" (which is independant of the selection). These Filter by options can be chosen in conjunction (And) resulting in a comparison where the selections appeared simultaneously. E.g. "Scope: Characters, Filter by: Season 8, Voice ActorZZ" will only tally episodes in season 8 where Voice ActorZZ was listed along with the character appearing in it.
